# KCIASSO — исправление прокрутки modal «Выбор размещений»

Дата: 2026-07-16.

## Цель

Исправить только прокрутку modal «Выбор размещений», предварительно подтвердив причину в реальном браузере. Permissions, backend source, Prisma, API, Kubb и business logic не изменялись.

## Диагностика до изменений

Проверка выполнена в Chromium через Playwright на уже работавших пользовательских KCIASSO frontend/backend (`3000`/`4000`). Процессы использовались только read-only, не перезапускались и не останавливались.

Сценарий:

1. Выполнен вход в admin UI.
2. Открыты «Документы» → «Добавить документ» → «Изменить размещение».
3. В modal раскрыта группа «Качество образования» с 31 пунктом.
4. Сняты DOM dimensions и computed styles overlay, dialog, общего списка, раскрытой группы и footer.

Исходные метрики на 1366×768:

- предполагаемый scroll area `.placementGroups`: `clientHeight=494`, `scrollHeight=494`, `overflow-y=auto`, `min-height=0`, computed height `494.25px`, `max-height=none`;
- раскрытая `.placementGroup`: `clientHeight=87`, `scrollHeight=1541`, computed height `88.6875px`, `overflow-y=hidden`, `max-height=none`;
- dialog: flex-column, `clientHeight=744`, computed height/max-height `744px`, без фиксированной `height` в CSS;
- footer был видим, но содержимое группы обрезалось;
- body имел `overflow:hidden`;
- горизонтального document overflow не было.

Исходные browser regression tests подтвердили баг на обоих viewport:

- 1366×768: `scrollHeight=494`, `clientHeight=494`;
- 390×844: `scrollHeight=526`, `clientHeight=526`.

Обе проверки закономерно упали на условии `scrollHeight > clientHeight`.

## Точная причина

Общий список был CSS Grid внутри flex dialog. При ограничении modal по `max-height` grid распределял доступную высоту между implicit rows и сжимал раскрытую строку группы примерно до 87 px. Одновременно `.placementGroup { overflow: hidden; }` обрезал её реальное содержимое высотой 1541 px.

Из-за clipping переполнение раскрытой группы не участвовало в scrollable overflow родительского `.placementGroups`: у общего списка получалось `scrollHeight == clientHeight`, поэтому wheel и изменение `scrollTop` не могли прокручивать список.

Проблема не была вызвана body lock, `preventDefault`, touch events или portal events. Глобальный `preventDefault` не используется; успешный wheel test после исправления это дополнительно подтверждает.

## Исправление

- Overlay приведён к фиксированной grid-структуре с `padding: 12px` и `overflow: hidden`.
- Dialog получил обязательную фиксированную высоту `min(860px, calc(100dvh - 24px))` и grid rows `auto auto minmax(0, 1fr) auto`.
- Единственным scroll container остаётся `.placementGroups` между search и footer.
- Для implicit rows списка установлено `grid-auto-rows: max-content`, поэтому раскрытая группа сохраняет естественную высоту.
- У `.placementGroup` удалён собственный `overflow:hidden`; фиксированного `max-height` у accordion panel нет.
- Header, search и footer находятся в отдельных grid rows и не прокручиваются.
- Существующая логика одной одновременно раскрытой группы и body lock не изменялась.
