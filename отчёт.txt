# KCIASSO — исправление ошибки `prisma generate` в Docker build

Дата: 2026-07-18.

## Цель

Устранить ошибку backend deployment на шаге `RUN npx prisma generate` в root `Dockerfile`. Изменение ограничено Prisma generator output path; permissions, API, migrations, DTO business logic и runtime behavior не менялись.

## Ошибка и состояние до исправления

GitHub Actions падал на:

```text
Dockerfile:13
RUN npx prisma generate
process ... did not complete successfully: exit code: 1
```

В логе Prisma показывала два output path:

```text
output = "../node_modules/@prisma/client"
output = "../node_modules/.prisma/client"
```

Локальный `npx prisma generate` в существующем рабочем `node_modules` проходил, поэтому локальная проверка была недостаточной. Рабочий Docker image выполняет чистый `npm ci`, после чего Prisma видит настоящий package directory `node_modules/@prisma/client`.

## Воспроизведение причины

Создана task-only чистая копия backend без `node_modules`, `dist`, storage, uploads, `.env` и `.data`.

1. `npm ci --no-audit --no-fund` — PASS, 900 packages.
2. В чистой копии исходный `npx prisma generate` — FAIL с точным сообщением Prisma:

```text
Generating client into ...\node_modules\@prisma\client is not allowed.
This package is used by `prisma generate` and overwriting its content is dangerous.
Suggestion: replace output "../node_modules/@prisma/client" with "../node_modules/.prisma/client".
```

Это подтверждает, что причина — генерация Client непосредственно поверх установленного `@prisma/client` package directory. Текущий локальный `node_modules` скрывал проблему остаточным ранее сгенерированным содержимым.

## Исправление

В `prisma/schema.prisma` изменён только generator output:

```prisma
generator client {
  provider = "prisma-client-js"
  output   = "../node_modules/.prisma/client"
}
```

Это стандартный Prisma path. Импорты `@prisma/client` остаются без изменений: package forwarding автоматически использует `.prisma/client`.

Не изменялись:

- модели и поля Prisma;
- migrations и database contract;
- Nest controllers/services/guards;
- API/OpenAPI/Kubb;
- Docker runtime command и storage mounts;
- permissions и business logic.

## Проверки после исправления

### Основной backend

- `npx prisma generate`: PASS.
- `npm run build`: PASS.
- `npm test -- --runInBand`: PASS, 5 suites, 30/30 tests.
- `git diff --check`: PASS.

### Чистая task-only копия

- `npm ci --no-audit --no-fund`: PASS, 900 packages.
- `npx prisma generate` с исправленной schema: PASS.
- `npm run build`: PASS.

Первый build чистой копии до копирования одного исключённого при подготовке test copy source-подкаталога `src/system/documents/storage` дал ожидаемые TS2307 из-за ошибки самого test-copy filter, не проекта. После включения этого source-подкаталога build прошёл.

### Docker

Docker build локально не запускался: Docker daemon Desktop был выключен и endpoint `npipe:////./pipe/dockerDesktopLinuxEngine` недоступен. Поэтому требуемая финальная проверка Docker должна быть выполнена GitHub Actions/Dokploy после публикации изменения. Сама failing команда Prisma воспроизведена в чистом npm окружении и исправлена тем же output path.

## Изменённые файлы

- `D:\Desktop\dev\web\orders\kciasso-backend\prisma\schema.prisma` — единственная production source правка: output Prisma Client.
- `D:\Desktop\dev\web\orders\kciasso-backend\PROJECT_STATE.md` — актуализирован generated path и verification state.
- `D:\Desktop\dev\web\orders\kciasso-backend\отчёт.txt` — полностью перезаписан этим отчётом.

Generated Prisma Client в `node_modules/.prisma/client` пересоздан штатной командой и не является Git-изменением. Kubb generated files не изменялись.

## Процессы, порты и cleanup

- Dev/backend servers не запускались.
- Пользовательские процессы и порты не затрагивались.
- Docker daemon не запускался.
- Чистая test-only копия и временные файлы после проверки удалены по точным task-owned путям.
- Global kill, `git reset`, `git restore`, `git clean` и destructive операции над репозиторием/production storage не применялись.

## Итог и следующий шаг

Причина подтверждена чистым окружением, output path исправлен на `node_modules/.prisma/client`, локальная и clean-copy генерация/build/tests проходят. Следующий обязательный шаг — rerun backend Docker/GitHub Actions build; ожидается, что `RUN npx prisma generate` больше не будет падать.
