# Устранение React и Next Image предупреждений

## 1. Симптомы

На public HomePage наблюдались предупреждения React о `key` и Next Image о legacy `objectFit` и отсутствующем `sizes` у шести официальных логотипов. Дополнительно проверялись сообщения о CSS preload, HMR и React DevTools.

## 2. Источник missing key

Источник подтверждён чтением `src/widgets/pages/HomePage/HomePage.tsx`: выражение `homeSectionsOrder.map((key) => sectionRegistry[key])` возвращало элементы реестра без `key` на непосредственном уровне списка.

## 3. Исправление key

Список секций теперь возвращает `<Fragment key={key}>...</Fragment>`. Это сохраняет управляемый порядок секций и добавляет ключ именно непосредственному элементу результата `map`.

## 4. Компоненты с legacy objectFit

Проблемный вызов находился в официальных ресурсах HomePage: шесть элементов `officialResourceCards` использовали `<Image fill objectFit="cover">`. Источники изображений: `mok.webp`, `mprf.webp`, `fspnvsok.webp`, `fipi.webp`, `fct.webp`, `fioko.webp`.

## 5. Исправление object-fit

Удалён legacy prop `objectFit`. Поведение `cover` сохранено через CSS-класс `officialLogo` в `HomePage.module.scss` с `object-fit: cover`.

## 6. Расчёт и добавление sizes

Размеры выбраны по фактической сетке `.officialGrid`: 3 колонки по умолчанию, 2 при ширине до 1024px, 1 при ширине до 720px. Добавлено значение `sizes="(max-width: 720px) 100vw, (max-width: 1024px) 50vw, 33vw"`.

## 7. Проверка шести изображений

Dev и production browser smoke-проверки подтвердили для всех шести файлов на desktop и mobile: `complete=true`, `naturalWidth>0`, `object-fit=cover`, корректный `sizes`, ненулевые размеры DOM. Для lazy-loaded изображений тест прокручивал `#official-resources` перед проверкой.

## 8. CSS preload warning

В исходниках проекта не найден ручной `<link rel="preload" as="style">`. В проверенных production runtime сообщениях CSS preload warning не появился. Если он возникает только в dev, это сообщение Next/HMR о служебной загрузке стилей, а не дефект пользовательской разметки.

## 9. Dev-only информационные сообщения

В dev зафиксированы только ожидаемые `%cDownload the React DevTools...` и `[HMR] connected`. Они не подавлялись и не изменялись. Production этих сообщений не показал.

## 10. Lint
